### 障害検知フロー
#### GitHub Actionsの実行結果
パイプラインが失敗した際メールへの通知機能により即座にエンジニアへ通知
#### 状態監視
terraform planおよびapplyの実行ログを保存し予期せぬリソース変更が発生した場合はGitHubの通知とログから特定する

### ロールバック・リカバリ戦略
#### 修正の特定
障害原因となったコミットログおよびTerraformの実行ログから不正な変更箇所を特定
#### コードによる修正
git revertを用いて直前の正常に動作していたコミットまでコードを戻す
git push を実行し自動的にパイプラインを再走させる
#### 環境の再適用
パイプラインが正常終了することでインフラ構成が正常時の状態に自動的に再構築される

### 注意点
#### 手動操作の禁止
コンソール画面からの直接修正は行わない
すべてコードの修正を介して実施する
#### データ不整合の確認
必要に応じてRDSやS3のデータ整合性を別途確認する（Terraform管理外の影響範囲の調査）
